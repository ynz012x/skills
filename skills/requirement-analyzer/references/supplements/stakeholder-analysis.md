# 干系人分析技巧

干系人分析是价值需求的核心活动，帮助识别和理解与项目相关的所有人员和组织。

## 干系人识别

### 识别维度

| 维度 | 问题 | 示例角色 |
|------|------|----------|
| **谁使用** | 谁直接使用系统？ | 普通用户、操作员 |
| **谁管理** | 谁管理系统和数据？ | 管理员、运营人员 |
| **谁决策** | 谁做项目决策？ | 产品负责人、管理层 |
| **谁付费** | 谁支付项目费用？ | 客户、预算负责人 |
| **谁开发** | 谁构建系统？ | 开发、测试、运维 |
| **谁集成** | 系统与谁对接？ | 外部系统、合作方 |
| **谁监管** | 谁监督合规？ | 审计、法务、监管机构 |

### 识别方法

**1. 组织结构法**
从组织结构图出发，识别各部门相关人员。

**2. 流程追溯法**
沿业务流程识别各环节参与者。

**3. 洋葱图法**
从核心向外扩展：
```
          ┌─────────────────┐
          │    外部环境      │  监管、合作方
          │  ┌───────────┐  │
          │  │  组织环境  │  │  管理层、其他部门
          │  │ ┌───────┐ │  │
          │  │ │ 项目  │ │  │  项目团队
          │  │ │┌─────┐│ │  │
          │  │ ││系统 ││ │  │  直接用户
          │  │ │└─────┘│ │  │
          │  │ └───────┘ │  │
          │  └───────────┘  │
          └─────────────────┘
```

## 干系人分析框架

### 权力-利益矩阵

```
高权力 │ 重点管理      │ 密切合作
       │ (保持满意)    │ (积极参与)
       ├───────────────┼───────────────
       │ 最少精力      │ 保持沟通
       │ (监控即可)    │ (及时通知)
低权力 └───────────────┴───────────────
           低利益           高利益
```

### 分析维度

| 维度 | 说明 | 评估方式 |
|------|------|----------|
| **影响力** | 对项目决策的影响程度 | 高/中/低 |
| **利益** | 与项目结果的利益相关度 | 高/中/低 |
| **态度** | 对项目的支持程度 | 支持/中立/反对 |
| **期望** | 对项目的具体期望 | 文字描述 |
| **关注点** | 最关心的问题 | 功能/进度/成本/质量 |

## 干系人访谈技巧

### 访谈准备

1. 了解受访者背景
2. 准备访谈提纲
3. 预估访谈时间（通常30-60分钟）
4. 准备记录工具

### 访谈问题清单

**开场问题**：
- 请介绍一下您的日常工作？
- 您与这个项目的关系是什么？

**需求探索**：
- 您希望这个系统帮您解决什么问题？
- 目前工作中最大的痛点是什么？
- 您理想中的系统应该是什么样的？

**优先级确认**：
- 如果只能实现3个功能，您选哪3个？
- 哪些功能是必须有的，哪些是锦上添花？

**约束了解**：
- 有什么限制条件需要考虑？
- 有什么必须遵守的规定或流程？

**结束问题**：
- 还有什么我没问到但您想说的？
- 您推荐我还应该和谁聊聊？

### 访谈技巧

| 技巧 | 说明 |
|------|------|
| **开放式提问** | 避免是非题，鼓励详细描述 |
| **追问细节** | "能举个例子吗？" "具体是怎么操作的？" |
| **复述确认** | "我理解您的意思是...对吗？" |
| **保持中立** | 不评判，不引导答案 |
| **记录原话** | 用受访者的原话记录，避免过早解读 |

## 干系人沟通计划

### 沟通矩阵

| 干系人 | 信息需求 | 沟通方式 | 频率 | 负责人 |
|--------|----------|----------|------|--------|
| 项目发起人 | 进度、风险、重大决策 | 周报、面谈 | 每周 | PM |
| 产品负责人 | 需求、设计、变更 | 评审会、即时通讯 | 每日 | BA |
| 开发团队 | 需求详情、优先级 | 站会、文档 | 每日 | BA |
| 最终用户 | 进展、培训计划 | 邮件、演示 | 每月 | PM |

### 沟通原则

1. **及时性**：重要信息及时传达
2. **透明性**：主动分享进展和问题
3. **针对性**：根据受众调整内容和形式
4. **双向性**：鼓励反馈，认真倾听

## 冲突处理

### 常见冲突类型

| 冲突类型 | 示例 | 处理策略 |
|----------|------|----------|
| **需求冲突** | 不同用户要求互斥功能 | 明确优先级，寻找折中 |
| **资源冲突** | 需求超出预算或时间 | 分阶段实现，调整范围 |
| **利益冲突** | 部门间利益不一致 | 升级决策，寻求共赢 |
| **认知冲突** | 对问题理解不同 | 澄清事实，建立共识 |

### 处理步骤

1. **识别冲突**：明确冲突点和各方立场
2. **分析原因**：理解深层利益和关注点
3. **寻找共同点**：找到各方都能接受的目标
4. **提出方案**：设计多个可能的解决方案
5. **协商决策**：推动各方达成一致
6. **记录确认**：书面记录决策结果
