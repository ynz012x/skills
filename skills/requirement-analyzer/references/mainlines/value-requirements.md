# 价值需求主线

价值需求回答"为什么做"的问题，是需求分析的起点和方向指引。

## 核心理念

**需求 = 预期 - 现状**

价值需求关注：
- 干系人期望解决什么问题？
- 项目成功的标准是什么？
- 各方利益如何平衡？

## 目标/愿景分析

### 目标层次

```
战略目标（Why）
    ↓
战术目标（What）
    ↓
操作目标（How）
```

| 层次 | 关注点 | 示例 |
|------|--------|------|
| **战略目标** | 业务价值、竞争优势 | 提升市场份额、降低运营成本 |
| **战术目标** | 能力建设、流程改进 | 实现在线下单、自动化审批 |
| **操作目标** | 具体功能、性能指标 | 订单创建时间<3秒、支持1000并发 |

### 目标描述SMART原则

- **S**pecific（具体）：目标明确，不模糊
- **M**easurable（可测量）：有量化指标
- **A**chievable（可达成）：在能力范围内
- **R**elevant（相关）：与业务价值相关
- **T**ime-bound（有时限）：有明确的时间节点

**示例**：
- 模糊：提升用户体验
- SMART：在6个月内将订单转化率从2%提升到3%

### 目标分解步骤

1. 从干系人访谈中提取期望
2. 识别核心业务问题
3. 定义成功标准
4. 分解为可实现的子目标
5. 确定优先级

## 干系人分析

### 干系人识别

**常见干系人类型**：

| 类型 | 角色 | 关注点 |
|------|------|--------|
| **客户** | 甲方代表、采购方 | 投资回报、项目进度 |
| **用户** | 最终使用者 | 易用性、功能满足度 |
| **管理层** | 决策者、预算负责人 | 成本、风险、战略价值 |
| **开发团队** | 开发、测试、运维 | 技术可行性、工作量 |
| **运营方** | 客服、运营人员 | 运营效率、问题处理 |

### 权力-利益分析

```
        高权力
           │
    重点管理  │  密切合作
           │
  ─────────┼───────── 高利益
           │
    最少精力  │  保持沟通
           │
        低权力
```

| 象限 | 策略 |
|------|------|
| 高权力+高利益 | 密切合作，积极参与 |
| 高权力+低利益 | 重点管理，保持满意 |
| 低权力+高利益 | 保持沟通，及时通知 |
| 低权力+低利益 | 最少精力，定期更新 |

### 干系人冲突处理

当干系人需求冲突时：
1. 识别冲突点
2. 分析各方立场和利益
3. 寻找共同目标
4. 提出折中方案
5. 升级到更高层级决策

## 需求优先级评估

### MoSCoW方法

| 优先级 | 含义 | 占比建议 |
|--------|------|----------|
| **Must** | 必须有，核心功能 | 60% |
| **Should** | 应该有，重要功能 | 20% |
| **Could** | 可以有，锦上添花 | 15% |
| **Won't** | 本次不做，未来考虑 | 5% |

### Kano模型

| 类型 | 特征 | 示例 |
|------|------|------|
| **基本需求** | 没有会不满，有了是应该的 | 系统稳定性、基本安全 |
| **期望需求** | 有了满意，没有不满意 | 响应速度、功能完整 |
| **兴奋需求** | 有了很惊喜，没有也不会不满 | 智能推荐、个性化 |

### 优先级评估维度

| 维度 | 权重建议 | 评估问题 |
|------|----------|----------|
| 业务价值 | 40% | 对业务目标的贡献度？ |
| 用户影响 | 30% | 影响多少用户？影响程度？ |
| 实现成本 | 20% | 开发工作量？技术风险？ |
| 时间紧迫 | 10% | 有无截止日期？依赖关系？ |

## 价值验证

### 验证清单

- [ ] 项目目标清晰、可测量
- [ ] 关键干系人已识别
- [ ] 干系人期望已记录
- [ ] 利益冲突已识别并有处理方案
- [ ] 需求优先级已评估
- [ ] 成功标准已定义
- [ ] 项目范围边界清晰

### 常见问题

| 问题 | 表现 | 解决方案 |
|------|------|----------|
| 目标不清 | 无法回答"为什么做这个项目" | 重新访谈关键干系人 |
| 范围蔓延 | 需求不断增加，无法收敛 | 明确优先级，坚守边界 |
| 利益冲突 | 干系人需求互相矛盾 | 升级决策，寻求平衡 |
| 期望过高 | 目标超出技术或资源能力 | 分阶段实现，管理预期 |
