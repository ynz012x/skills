# 阶段1检查清单：明确目标和范围

在进入阶段2之前，请确认以下检查项全部通过。

---

## 干系人分析

- [ ] 已识别所有关键干系人类型（至少包含3类）
  - [ ] 客户/甲方代表
  - [ ] 最终用户（按角色分类）
  - [ ] 管理层/决策者
  - [ ] 开发团队
  - [ ] 运维团队
  - [ ] 外部系统/合作方

- [ ] 每个干系人的基本信息已记录
  - [ ] 角色/职位
  - [ ] 联系方式
  - [ ] 所属部门/组织

- [ ] 每个干系人的分析维度已评估
  - [ ] 影响力（高/中/低）
  - [ ] 利益相关度（高/中/低）
  - [ ] 态度（支持/中立/反对）

- [ ] 干系人期望和关注点已明确记录

- [ ] 存在利益冲突的干系人已识别，并有初步处理方案

- [ ] 干系人沟通计划已制定

---

## 目标与范围

- [ ] 项目目标清晰、可衡量（符合SMART原则）

- [ ] 项目愿景已定义

- [ ] 系统边界明确
  - [ ] 哪些功能在范围内
  - [ ] 哪些功能在范围外
  - [ ] 与外部系统的边界

- [ ] 项目约束条件已明确
  - [ ] 时间约束
  - [ ] 预算约束
  - [ ] 技术约束
  - [ ] 法规约束

- [ ] 成功标准已定义

- [ ] 需求优先级评估方法已确定

---

## 主题域划分（SERU-S）

- [ ] 主题域数量合理（3-8个）

- [ ] 每个主题域有明确的业务职责

- [ ] 主题域之间边界清晰，无职责重叠

- [ ] 主题域覆盖所有业务范围，无遗漏

- [ ] 每个主题域的核心职责已描述

- [ ] 主题域之间的关系已识别

- [ ] 主题域关系图已绘制

---

## 业务事件（SERU-E）

- [ ] 已识别主要业务事件（至少5个）

- [ ] 事件已分类
  - [ ] 外部事件
  - [ ] 时间事件
  - [ ] 状态事件

- [ ] 每个事件的触发者已明确

- [ ] 事件的频率已评估（高/中/低）

- [ ] 事件的重要性已评估（高/中/低）

- [ ] 事件已按主题域归类

---

## 报表与管控点（SERU-R）

- [ ] 已识别关键输出报表

- [ ] 报表的使用者和用途已明确

- [ ] 报表的更新频率已确定

- [ ] 管控点已识别（审批点、检查点、监控点）

- [ ] 管控点的触发条件已明确

---

## 文档输出

- [ ] 主题域清单和关系图已输出

- [ ] 报表和管控点清单已整理
