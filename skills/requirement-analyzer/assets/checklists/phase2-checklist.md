# 阶段2检查清单：理清脉络和框架

在进入阶段3之前，请确认以下检查项全部通过。

---

## 领域模型

### 实体识别

- [ ] 核心实体已识别（建议10-30个）

- [ ] 每个实体有明确的业务定义

- [ ] 实体类型已区分
  - [ ] 实体（有唯一标识，独立生命周期）
  - [ ] 值对象（无唯一标识，依附于实体）

- [ ] 每个实体的关键属性已列出（3-5个核心属性）

- [ ] 属性的数据类型已初步定义

### 实体关系

- [ ] 实体之间的关系已明确

- [ ] 关系类型已确定
  - [ ] 一对一
  - [ ] 一对多
  - [ ] 多对多

- [ ] 关系强度已评估
  - [ ] 组合（整体删除时部分也删除）
  - [ ] 聚合（整体删除时部分可独立存在）
  - [ ] 关联（双方独立存在）

- [ ] 关系的业务含义已描述

### 模型验证

- [ ] 领域模型已与业务专家验证

- [ ] 核心实体覆盖所有主题域

- [ ] 没有孤立的实体（每个实体至少有一个关系）

- [ ] 领域模型图（ER图）已绘制

---

## 用例模型（SERU-U）

### 参与者识别

- [ ] 参与者（Actor）已完整识别
  - [ ] 人员角色
  - [ ] 外部系统
  - [ ] 定时器

- [ ] 每个参与者有明确的说明

- [ ] 参与者与主题域的关系已明确

### 用例识别

- [ ] 用例已从业务事件转化

- [ ] 用例覆盖所有主要业务事件

- [ ] 用例粒度合理
  - [ ] 不过大（包含多个独立业务目标）
  - [ ] 不过小（无法独立产生价值）

- [ ] 用例命名规范（动宾短语）

### 用例分类

- [ ] 用例已分类
  - [ ] 业务支持功能（60-70%）
  - [ ] 管理支持功能（20-30%）
  - [ ] 维护支持功能（10-15%）

- [ ] 每个主题域至少有3-5个用例

### 用例关系

- [ ] 用例之间的关系已识别
  - [ ] 包含关系（include）
  - [ ] 扩展关系（extend）
  - [ ] 泛化关系

### 用例概要

- [ ] 每个用例有概要描述
  - [ ] 参与者
  - [ ] 前置条件
  - [ ] 后置条件
  - [ ] 主成功场景（简要）

- [ ] 用例图已绘制
