## 一致性检查

### 模型一致性

- [ ] 领域模型与用例模型一致
  - [ ] 用例中使用的实体都在领域模型中
  - [ ] 领域模型中的实体都有对应的用例操作

- [ ] 用例与业务事件一致
  - [ ] 每个业务事件有对应的用例
  - [ ] 用例覆盖所有业务场景

### 规则一致性

- [ ] 业务规则与用例一致
  - [ ] 用例中引用的规则都已定义
  - [ ] 规则的适用范围与用例匹配

- [ ] 非功能需求与用例一致
  - [ ] 特殊需求在相关用例中已标注

### 术语一致性

- [ ] 术语使用一致
  - [ ] 同一概念使用同一术语
  - [ ] 术语表已更新

---

## 可追溯性检查

### 需求追溯

- [ ] 每个用例可追溯到业务事件或干系人需求

- [ ] 每个非功能需求可追溯到质量目标

- [ ] 每个业务规则可追溯到业务需求

### 追溯矩阵

- [ ] 需求追溯关系可建立（如需要）
