# Step 4 检查清单：分层架构决策

在进入 Step 5 之前，确认以下产出完整：

## 必须项

- [ ] 架构风格已选择（Clean / Hexagonal / Vertical Slice / 混合）
- [ ] 选择理由已记录为 ADR 格式
- [ ] 分层结构图已绘制（ASCII 或 Mermaid）
- [ ] 项目目录结构已生成（完整 ASCII 树）
- [ ] 目录结构中各层/模块的职责已标注

## standard/comprehensive 额外项

- [ ] 跨切面关注点策略已确定
  - [ ] 异常处理策略
  - [ ] 日志规范
  - [ ] 认证鉴权位置
  - [ ] 数据验证层级
  - [ ] 事务管理方式
- [ ] 依赖注入策略已确定
- [ ] 模块间依赖方向已标注
- [ ] 备选架构的权衡已记录

## 完整性验证

确认以下问题已有答案：
1. 依赖方向是否一致？是否存在反向依赖？
2. 领域层是否独立于框架和基础设施？
3. 目录结构是否支持团队并行开发？
4. 新功能添加时修改哪些文件是否清晰？
5. 单元测试时是否可以独立测试领域逻辑（无需启动框架）？
6. 目录结构是否与 Step 2 的 BC 划分一致？
