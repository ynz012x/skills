# Step 3 检查清单：战术设计

在进入 Step 4 之前，确认以下产出完整：

## 必须项

- [ ] 核心 BC 的聚合已识别（至少列出聚合根名称）
- [ ] 每个聚合遵循 Vernon 四规则验证
  - [ ] 在事务边界内保护业务不变量
  - [ ] 聚合尽量小
  - [ ] 通过唯一标识引用其他聚合
  - [ ] 使用最终一致性跨聚合边界
- [ ] 实体与值对象已区分
- [ ] 聚合结构图已绘制（Mermaid classDiagram）

## standard/comprehensive 额外项

- [ ] 领域事件已设计（事件名、负载、触发条件）
- [ ] 领域服务已识别（无法归属于单个聚合的操作）
- [ ] Repository 接口已定义（每个聚合根一个）
- [ ] Factory 使用场景已确定（复杂创建逻辑）
- [ ] 聚合间协作的事件流图已绘制
- [ ] 值对象的相等性判断标准已明确

## 完整性验证

确认以下问题已有答案：
1. 每个聚合的事务边界是否清晰？是否存在跨聚合事务？
2. 聚合是否足够小？加载一个聚合是否会连带大量数据？
3. 聚合间的引用是通过 ID 还是直接对象引用？
4. 跨聚合的业务规则如何保证一致性（Saga/事件补偿）？
5. 是否存在"上帝聚合"（聚集了过多职责）？
