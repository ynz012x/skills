# Step 2 检查清单：战略设计

在进入 Step 3 之前，确认以下产出完整：

## 必须项

- [ ] 统一语言术语表已建立（至少覆盖核心业务概念）
- [ ] 限界上下文已划分（至少 2 个 BC）
- [ ] 每个 BC 有明确的职责描述（1-2 句话）
- [ ] Context Map 已绘制（Mermaid 图）
- [ ] BC 间关系类型已标注（U/D、Shared Kernel、ACL 等）
- [ ] 域分类已完成（核心域/支撑域/通用域）

## standard/comprehensive 额外项

- [ ] Event Storming 关键事件已识别
- [ ] 每个 BC 的输入/输出边界已明确
- [ ] BC 与团队映射关系已考虑
- [ ] 通用域的复用策略已确定（自建/购买/开源）
- [ ] 核心域的竞争优势已说明

## 完整性验证

确认以下问题已有答案：
1. 每个 BC 内的术语是否唯一、无歧义？
2. 同一概念在不同 BC 中的含义差异是否已记录？
3. BC 间的集成方式是否明确（同步/异步/事件驱动）？
4. 哪个 BC 是核心域？投入最多资源建设的应该是哪个？
5. Context Map 中是否存在高耦合关系需要解耦？
